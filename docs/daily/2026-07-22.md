### 2. 优化前端资源更新与异常恢复

- 新增 `src/utils/appReload.ts`，识别动态导入失败、chunk 加载失败、CSS preload 失败等典型前端资源更新异常。
- `src/components/ErrorBoundary.tsx` 接入资源更新识别逻辑，遇到旧资源导致的页面异常时自动触发受控刷新。
- 刷新逻辑增加 URL 参数和 `sessionStorage` 冷却时间，避免资源异常时无限刷新。
- `src/main.tsx` 在应用启动时清理刷新参数，减少用户看到内部参数的概率。
- `src/App.tsx` 清理已不再需要的页面引用，使页面路由结构更聚焦。

### 3. 调整 Nginx 与 Docker 部署策略

- `nginx.conf` 将 `/assets/` 下带 hash 的构建产物改为命中即长期缓存、缺失即返回 404，避免旧 HTML 将不存在的 JS chunk 回退成 `index.html`。
- `nginx.conf` 对 SPA 业务路由保留 `index.html` 回退，并给入口 HTML 增加不缓存策略，确保部署后浏览器能拿到最新资源清单。
- `Dockerfile` 和 `server/Dockerfile` 的依赖安装增加 `--no-audit --no-fund`，减少部署构建阶段受 npm 审计网络请求影响。
- `Dockerfile`、`docker-compose.yml`、`server/Dockerfile` 移除或弱化镜像/Compose 层 healthcheck 依赖，改由部署脚本做更明确的启动验证。
- `docker-compose.yml` 将默认 `ADMIN_USERNAME` 调整回 `Mixmind`，并简化服务依赖关系，降低 Compose 条件健康检查导致的启动阻塞。

### 4. 强化服务器同步与上线脚本

- `sync-to-server.sh` 重构为更完整的远程部署流程：打包、上传、远程备份、同步文件、补齐 `.env`、重建服务、健康检查。
- 脚本会自动生成缺失或不安全的 `JWT_SECRET`、`REFRESH_TOKEN_SECRET`、`DB_PASSWORD`、`ADMIN_PASSWORD` 等关键密钥，降低部署因默认弱口令失败或遗留配置错误的风险。
- 远程部署时先启动 PostgreSQL，循环等待数据库可用，再同步数据库用户密码，最后构建并启动后端与前端。
- 部署完成后分别验证后端健康接口和前端页面，并在失败时输出最近容器日志，便于定位问题。
- 同步时排除 `.env`、`backups/`、`logs/`、临时远程脚本等文件，减少误覆盖和敏感文件泄露风险。

### 5. 改进批量导入与数据覆盖逻辑

- `server/src/routes/data.ts` 为各类数据模型补充唯一键定义，批量导入从单纯 `createMany` 调整为事务内按业务唯一键 `upsert`。
- 导入时同一唯一键已存在的数据会被更新，不再因为重复数据直接失败，也更符合“覆盖导入”的业务预期。
- `src/components/data/ImportDialog.tsx` 增加覆盖月份选择，用户可按导入文件中识别出的月份勾选本次要导入/更新的数据范围。
- 导入弹窗明确展示唯一键口径、有效行数、错误分类、跳过错误行选项和最终导入结果，降低误导入风险。
- `server/src/schemas/data.ts` 配合前端字段规则做校验调整，支持更贴近实际业务数据的导入格式。

### 6. 调整薪资与字段标准口径
