## 二、完成事项

### 1. 完成全面优化计划

- 新增 `docs/optimization-plan.md`，基于项目文件审查梳理出高优先级风险、性能问题和可维护性问题。
- 将问题按 P0、P1、P2 分层，明确哪些属于上线前必须处理，哪些属于下一版本和长期优化。
- 覆盖领域包括生产部署、数据库迁移、权限安全、数据校验、看板性能、巨组件拆分和文档维护。

### 2. 落地账号与权限体系

- 建立 `super_admin`、`hr_admin`、`hr_staff`、`finance`、`dept_manager`、`auditor` 六类角色的统一权限口径。
- 新增账号管理能力，支持账号列表、账号详情、创建账号、编辑账号、重置密码和停用账号。
- 前端新增账号管理页面，并在侧边栏和顶部用户信息中集成当前用户权限与身份展示。
- 新增 `docs/account-permissions.md`，沉淀角色权限矩阵、数据写入范围、账号管理 API 示例和安全限制。

### 3. 强化后端安全与权限校验

- 登录和 Token 校验阶段会拒绝无效角色或已停用账号。
- 限制 `hr_admin` 管理 `super_admin`，避免越权创建、修改或停用超级管理员。
- 禁止停用当前登录账号，并保证系统至少保留一个启用状态的 `super_admin`。
- 在角色变更、账号停用、密码重置后撤销 refresh token，降低旧凭证继续可用的风险。
- 后端引入 Helmet、接口限流、生产错误脱敏和认证用户短缓存。

### 4. 优化数据接口与计算链路

- 新增 `server/src/schemas/data.ts`，通过 Zod 对后端数据写入进行统一校验。
- 批量导入增加 5000 条上限，并优化为 `createMany` 批量写入。
- 数据查询对合法角色开放，新增、导入、编辑、删除和全量清空按角色做分级控制。
- `server/src/services/salaryCalculator.ts` 从全表扫描调整为定向重算当前月、下月和下一年同月，减少不必要计算。
- 审计日志补充真实表名、脱敏后的变更内容和新增记录 ID，提升问题追踪能力。

### 5. 改进生产部署与迁移流程

- 后端容器启动从 `prisma db push` 调整为 `prisma migrate deploy`，减少生产环境 schema 漂移和误改风险。
- 新增 Prisma 初始化迁移文件 `server/prisma/migrations/20260721130000_init/migration.sql`。
- 部署文档补充历史 `db push` 数据库的 baseline 处理方式。
- 清理默认 `Mixmind / Mixmind` 凭据，部署脚本改为要求强密钥、显式服务器和部署前备份。
- 更新 `deploy.sh`、`sync-to-server.sh`、`scripts/ubuntu-docker-deploy.sh` 等部署相关脚本。

### 6. 优化前端体验与结构

- `src/App.tsx` 增加页面懒加载，降低初始加载压力。
- `src/lib/api.ts` 增加 Token 自动刷新逻辑，减少会话过期导致的中断体验。
- 新增看板聚合加载能力，降低前端一次性发起过多请求的压力。
- `src/components/dashboard/useDashboardLayout.ts` 修复异步布局水合覆盖本地操作的问题。
- `src/pages/DashboardPage.tsx`、`src/components/layout/Sidebar.tsx`、`src/components/layout/Header.tsx` 根据用户权限展示不同入口。
