# 已知问题列表

## 1. 数据存储限制

- **问题**：所有业务数据存储在浏览器 localStorage，单域限制约 5-10MB。
- **影响**：大量历史数据累积后可能超出存储上限。
- **规避**：定期导出数据备份（DataContext 已实现 `exportData()`），或对接后端 API。
- **状态**：设计中（预留了 API 环境变量配置点位）。

## 2. 多用户/多设备数据不共享

- **问题**：localStorage 是按浏览器/设备隔离的，不同用户、不同设备之间数据独立。
- **影响**：无法实现多人协作编辑，每个用户看到的数据不同。
- **规避**：如需多人共享，必须引入后端数据库 + API。
- **状态**：当前架构限制。

## 3. 认证为前端 Mock

- **问题**：登录使用硬编码账号 `admin/admin`，`useAuth.ts` 仅在浏览器内存中管理登录状态。
- **影响**：缺乏真正的身份验证和权限控制。
- **规避**：生产环境需替换为实际认证服务（预留了 `VITE_AUTH_API_URL` 配置）。
- **状态**：待对接。

## 4. 数据迁移需手动触发

- **问题**：成本构成（CostComposition）字段名从旧版本迁移到新版本，由 `DataContext.loadFromStorage` 自动执行。但如果备份数据未迁移，可能需要手动清除缓存后重新加载。
- **影响**：固浮比报表显示全零。
- **规避**：
  1. 打开浏览器开发者工具 (F12) → Application → Local Storage
  2. 删除 `salary-admin-data` 和 `salary-admin-data-backup` 两个 key
  3. 刷新页面，系统自动加载默认模拟数据
  4. 重新编辑数据
- **状态**：已添加自动迁移逻辑（2026-07-10），首次加载自动转换。

## 5. 浏览器兼容性

- **确认支持**：Chrome/Edge >= 100、Firefox >= 100、Safari >= 15
- **不支持**：IE 11 及以下版本
- **可能异常**：部分国产浏览器（360 安全浏览器等）的隐私模式下可能禁用 localStorage

## 6. 构建产物较大

- **问题**：生产构建的 JS bundle 约 1.18 MB (gzip 约 332 KB)，主要来自 recharts 图表库。
- **影响**：首次加载较慢（尤其在移动网络）。
- **优化方案**：可考虑代码分割（Code Splitting），按页面懒加载 recharts。
- **状态**：已标注待优化。

## 7. CSV 导入编码问题

- **问题**：CSV 导入支持 UTF-8 和 GBK 编码自动检测，但部分 Excel 导出的 CSV 文件可能有 BOM 头或混合编码导致解析异常。
- **影响**：导入时中文乱码或字段解析错误。
- **规避**：使用「另存为 CSV UTF-8」格式导出。

## 8. 公式引擎安全性

- **问题**：`formulaEngine.ts` 使用 `new Function()` 计算表达式，存在代码注入风险。
- **影响**：如果恶意用户在字段配置中插入恶意代码，可能被执行。
- **缓解**：字段配置页面仅管理员可见，且公式表达式经过正则校验（仅允许数字/运算符/字段名）。
- **状态**：安全沙箱可用但非绝对安全。

## 9. 端口冲突

- **问题**：Docker 部署默认使用 80 端口，开发服务器使用 5173 端口。
- **影响**：如果端口被占用，需要手动修改。
- **规避**：Docker 部署时修改 `-p 8080:80`；开发时使用 `npm run dev -- --port 3000`。

## 10. 总览维度成本构成无数据

- **问题**：数据看板「总览」维度的「人力成本构成」图表，因 compositions 数据中没有 `department === '全公司'` 的记录，会显示为空。
- **影响**：切换到「总览」维度时，饼图为空。
- **规避**：切换到具体板块（总部/自营等）查看对应成本构成。
- **状态**：设计如此 — 总览展示的是一级板块概览 KPI，明细图表需进入各板块查看。
