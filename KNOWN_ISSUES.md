# 已知问题列表

本文档仅列出当前版本仍存在的限制与风险；已在 v1.2 解决的问题保留在文末，避免排障时混淆现状。

## 当前问题

### 1. 数据迁移需在部署前明确处理

- **问题**：系统已由 `prisma db push` 切换为 `prisma migrate deploy`。已有生产数据库在首次采用迁移时，需要先按 `DEPLOY.md` 标记初始迁移，不能直接重复建表。
- **影响**：跳过基线步骤会导致首次部署迁移失败。
- **规避**：先完成数据库备份，再执行 `npx prisma migrate resolve --applied 20260721130000_init`，之后才使用 `migrate deploy`。

### 2. 金额字段仍使用 PostgreSQL `double precision`

- **问题**：现有 Schema 的金额列仍为 Prisma `Float`，高频累加计算可能出现二进制浮点精度误差。
- **影响**：极端情况下报表汇总可能出现小数尾差。
- **规避**：展示层统一按两位小数处理；下一次涉及数据库迁移的版本应改为 `Decimal(18, 2)` 或以分为单位的整数。

### 3. 浏览器兼容性

- **确认支持**：Chrome/Edge >= 100、Firefox >= 100、Safari >= 15。
- **不支持**：IE 11 及以下版本。
- **可能异常**：部分国产浏览器的隐私模式可能限制本地存储或下载功能。

### 4. 构建产物较大

- **问题**：图表库占用较多初始 JavaScript 体积。
- **影响**：弱网环境下首次加载较慢。
- **状态**：已通过页面懒加载降低首屏开销；如仍需优化，可评估以 ECharts 替换 Recharts。

### 5. CSV 导入编码问题

- **问题**：尽管已支持 UTF-8/GBK 自动识别，部分 Excel 导出的混合编码 CSV 仍可能解析异常。
- **规避**：使用「另存为 CSV UTF-8」格式导出；导入前先在预览中确认字段映射。

### 6. 公式引擎安全性

- **问题**：公式表达式须经过严格白名单校验，但复杂公式能力仍有限。
- **影响**：不支持任意 JavaScript 或自定义函数。
- **状态**：这是有意的安全限制；如需扩展，请采用受控表达式解析器而非执行动态代码。

### 7. 端口冲突

- **问题**：Docker 默认映射 80 端口，开发服务器默认使用 5173 端口。
- **规避**：在 `.env` 中设置 `CLIENT_PORT`，或使用 `npm run dev -- --port <端口>`。

### 8. 总览维度成本构成无数据

- **问题**：数据看板「总览」维度的「人力成本构成」图表没有 `department === '全公司'` 的组成记录时会显示为空。
- **影响**：仅影响总览下的明细图表，不影响具体板块的数据。
- **规避**：切换到具体板块查看，或补录「全公司」成本构成数据。

## v1.2 已解决

### ✅ 业务数据仅存于浏览器 localStorage

- **原问题**：数据受浏览器容量限制，无法可靠持久化。
- **解决**：业务数据已迁移至 PostgreSQL，并通过后端 API 读写。

### ✅ 多用户/多设备数据不共享

- **原问题**：localStorage 按浏览器隔离，无法协作。
- **解决**：所有业务数据均由服务端数据库统一管理，已支持多账号权限控制。

### ✅ 前端 Mock 认证

- **原问题**：登录使用硬编码账号，缺乏真实鉴权。
- **解决**：已使用 JWT 双 Token、服务端账号管理、角色权限控制与停用账号 Token 撤销机制。
